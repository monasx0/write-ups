Meow

// First HTB challenge — telnet misconfiguration & default credentials

After spending time sharpening skills on TryHackMe CTFs, I decided to branch out and give HackTheBox a try. On my first day, the Meow challenge from the Starting Point appeared — an approachable beginner box designed to get you comfortable with the HTB environment.

This write-up covers the complete walkthrough: from setting up the VPN connection and running initial reconnaissance, to discovering a misconfigured Telnet service and catching the flag. Tools used: openvpn, ping, nmap, telnet.

guided questions
Q1What does the acronym VM stand for?
Virtual Machine
A software-based emulation of a physical computer. HTB uses VMs as both the attacker and target environment.
Q2What tool do we use to interact with the OS via command line — also called a console or shell?
Terminal
The terminal is where everything happens in CTFs — running tools, reading output, catching flags.
Q3What service do we use to form our VPN connection into HTB labs?
OpenVPN
Just like TryHackMe, HTB requires a VPN tunnel to reach lab machines. Download a .ovpn config file and connect using openvpn <file.ovpn> in your terminal.
Q4What tool do we use to test our connection to the target with an ICMP echo request?
Ping
Before running any heavy recon tools, a quick ping <target-ip> confirms the machine is alive and reachable.
Q5What is the most common tool for finding open ports on a target?
Nmap
The industry-standard network scanner. Used for discovering open ports, running services, OS fingerprinting, and much more.
Q6What service do we identify on port 23/tcp during our scans?
Telnet
Port 23 is the well-known port for Telnet — an older, unencrypted remote access protocol. Seeing it open is immediately a red flag in a CTF environment.
Q7What username is able to log into the target over Telnet with a blank password?
root
In misconfigured or default setups, the root account sometimes has no password set — giving direct administrative access. This is exactly the kind of misconfiguration that earns findings in real-world bug bounty programs too.
flag capture
⚑ Catching the flag

After confirming port 23 is open with nmap, we connect via Telnet using root with no password. Once inside the shell, the flag file is retrieved with a simple cat command.

$ nmap -sV <TARGET-IP>
PORT STATE SERVICE VERSION
23/tcp open telnet Linux telnetd

$ telnet <TARGET-IP>
Trying <TARGET-IP>...
Connected to <TARGET-IP>.
login: root
Password: [left blank — press Enter]
Welcome to Ubuntu...

root@Meow:~# ls
flag.txt
root@Meow:~# cat flag.txt
